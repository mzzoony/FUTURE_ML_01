# Contributing Guidelines

Thank you for your interest in this Machine Learning internship repository!

## How to Contribute

While this is primarily a personal internship repository, contributions in the form of suggestions, feedback, and improvements are welcome.

### Reporting Issues

If you find any issues or have suggestions:
1. Check if the issue already exists
2. Create a new issue with a clear description
3. Include relevant details (error messages, expected vs actual behavior)

### Suggesting Enhancements

Enhancement suggestions are welcome! Please:
1. Clearly describe the enhancement
2. Explain why it would be useful
3. Provide examples if applicable

### Code Contributions

If you'd like to contribute code:
1. Fork the repository
2. Create a new branch (`git checkout -b feature/improvement`)
3. Make your changes
4. Test your changes
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Add docstrings to functions and classes
- Include type hints where applicable
- Write clear, descriptive variable names
- Add comments for complex logic

### Commit Messages

Write clear commit messages:
- Use present tense ("Add feature" not "Added feature")
- Be descriptive but concise
- Reference issues when applicable

### Documentation

- Update README.md if needed
- Add comments to code
- Document new features or changes

## Questions?

Feel free to open an issue for any questions or clarifications.

Thank you for contributing!
